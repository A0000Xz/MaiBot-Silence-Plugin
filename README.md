**这是一个让麦麦更人性化保持沉默的插件，目前支持麦麦的0.12.2dev版！**

**过往版本不再提供文档指导，但过往版本仍然具备可用性**

**警告：使用此插件一定要看README.md最下方的须知，因为不看而可能导致的问题概不解释**

相信大家在使用麦麦的时候经常会遇到麦麦乱说话的问题，又或是回复频率过高对他人造成骚扰，再或者你也单纯看不下去想让麦麦保持沉默。

或者你也有想让麦麦只是看着群里人聊天，纯窥屏就是不说话的小众需求。亦或者是你不希望麦麦在被禁言后还在思考，试图发消息浪费Token。

于是，本插件诞生了，致力于在围绕“沉默”的主题下，提供许多实用性功能。

只要你的麦麦人设相对正常，你只需要对她礼貌的说一句“请安静”或者“请安静五分钟”之类的话，她便会乖乖听话安静。

你可以用自然语言指定她保持沉默的时间，也有一个配置项控制了通过自然语言能触发的沉默时间上限（为了防止有坏人让麦麦闭嘴太长时间）

通过正在测试的实验性功能配置项，你也可以配置麦麦默认对哪些人或者哪些群始终保持沉默，而且在这种情况下麦麦属于是窥屏状态，消息照收不误，依然能学习表达方式，但就是不会进行多余的决策来浪费你的Token。

当然，如果你的麦麦使用的模型足够聪明的话，她应该也会看聊天氛围选择主动保持沉默一段时间。如果你想要麦麦立刻回来的话，艾特她就可以了。

此外，正如上述所言，该插件会自动在麦麦被禁言，或者群聊被全体禁言时让麦麦保持沉默，Save your Token!

本插件提供了很多配置选项，推荐使用WebUI进行配置！

本插件也提供了一些指令以方便强制性操作，目前有的指令如下：

/silence true <times> 立刻在你发出这条指令的聊天环境内让麦麦陷入被指定的沉默状态，如果不提供后面的时间则为强力的永久沉默（无法被艾特打断），提供的时间必须以秒为单位，例如：

*"/silence true 114514" ———— 让麦麦保持沉默114514秒*

/silence false 立刻在你发出这条指令的聊天环境内让麦麦退出沉默状态。

插件也提供了权限控制配置项，确保只有指定的人能够使用指令。

**须知：**

**1，本插件的优先级很高，可能会干预其他插件的运作，可能会出现无法预料的兼容性问题（截至目前的测试未遇到），有问题可以积极与作者在麦麦技术群联系或提交issue。**

**2，本插件在沉默期间默认会中止一切的决策行为，其他Action类,Tool类插件的一切行为也会因此被中止。（关于其他插件的命令系统如何受影响是可以被配置的）**

**3，如果你的麦麦在运行过程中被关闭的话，重启后默认解除已记录的沉默状态。**

**4，沉默插件并不能阻止已经在进行的一些事进行，例如其他插件添加的已经在运行的定时任务，正在执行的Action或者Command等。**

**5，这个插件被刻意设计了不会在私聊环境生效，就算生效也有保险措施以确保不会进入沉默。**

**6，沉默状态的优先级为 自然语言 < 命令指定 < 实验性功能配置，优先级更高的效果会覆盖优先级较低的效果，例如指令可以任意修改当前的沉默状态，但是不能解除实验性功能里设置的默认沉默。**

**7，指令的效果对于低于自身优先级的状态是绝对覆盖的，但请记住，如果指令指定的不是永久沉默，艾特也是可以打断沉默的。**

**8，由于适配器没有提供禁言状态的实时更新接口，因此请确保你的麦麦在被人禁言和解除禁言的期间不要重启，重启会丢失已有的沉默状态，不过你也可以自己用指令补上就是。**

**9，由于沉默插件主要依靠EventHandler实现核心功能，因此可能与其他EventHandler类插件存在优先级冲突，请在挑选插件时注意。**
